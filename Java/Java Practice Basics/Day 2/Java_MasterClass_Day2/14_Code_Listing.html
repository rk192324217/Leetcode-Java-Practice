1. Multiplication Table (13)

2. Sum of Digits (1234 → 10)

int n=1234, s=0;
while(n>0) {
    s += n%10;
    n /= 10;
}
System.out.println("Sum: " + s);

💡 Logic: What happens?

Extract last digit using %10, adds to sum, then removes last digit using /10 until n reaches 0.

❓ Requirement Change: What if...?

Change: Only sum EVEN digits.
Do: Wrap s += n%10 in an if((n%10)%2 == 0) block.

🕒 Session 2: Loop Problems

4. Print Primes (1 to 100)

for (int n = 2; n <= 100; n++) {
    boolean isPrime = true;
    for (int i = 2; i <= Math.sqrt(n); i++) {
        if (n % i == 0) { isPrime = false; break; }
    }
    if (isPrime) System.out.print(n + " ");
}

💡 Logic: What happens?

Nested loops: Outer selects candidate, inner checks divisors up to √n. break optimizes for composite numbers.

🔍 See Visual Proof: Why √n?

❓ Requirement Change: What if...?

Change: Find first 5 prime numbers only.
Do: Add a counter variable; if(count == 5) break; in outer loop.

5. FizzBuzz (1-50)

for (int i = 1; i <= 50; i++) {
    if (i % 3 == 0 && i % 5 == 0) System.out.println("FizzBuzz");
    else if (i % 3 == 0) System.out.println("Fizz");
    else if (i % 5 == 0) System.out.println("Buzz");
    else System.out.println(i);
}

💡 Logic: What happens?

Priority checking! Must check 15 (3 & 5) first, otherwise 3/5 will "catch" it first and skip "FizzBuzz".

❓ Requirement Change: What if...?

Change: Instead of 1, start from index 50 to 1.
Do: for(int i=50; i>=1; i--).

6. Reverse a Number (1234 → 4321)

int n = 1234, rev = 0;
while (n != 0) {
    rev = rev * 10 + (n % 10);
    n /= 10;
}
System.out.println("Reversed: " + rev);

💡 Logic: What happens?

Build reversed number digit-by-digit. *10 shifts left, %10 adds newest digit from right of n.

❓ Requirement Change: What if...?

Do: Output 21 (int) or logic to print 021 (string builder).

Loop uses cumulative product. Recursion breaks problem into 5 * (4 * (3...)). Base case 0! = 1 is critical.

🕒 Session 3: Nested Loops & Patterns

8. Right-angled Star Triangle (5 rows)

for (int i = 1; i <= 5; i++) {
    for (int j = 1; j <= i; j++) {
        System.out.print("* ");
    }
    System.out.println();
}

💡 Logic: What happens?

Outer loop i controls rows (1 to 5). Inner loop j prints stars based on current row number (Row 1 gets 1 star, etc.).

❓ Requirement Change: What if...?

Change: Print numbers instead of stars (1 / 2 2 / 3 3 3).
Do: Change System.out.print("* ") to System.out.print(i + " ").

🚀 Try Code

9. Inverted Triangle Pattern

for (int i = 5; i >= 1; i--) {
    for (int j = 1; j <= i; j++) {
        System.out.print("* ");
    }
    System.out.println();
}

💡 Logic: What happens?

Exact opposite of the standard triangle. Row count starts at 5 and decreases. Inner loop follows i.

❓ Requirement Change: What if...?

Change: Add leading spaces to make it right-aligned.
Do: Add a third loop before the star loop to print 5-i spaces.

🚀 Try Code

10. Number Pyramid (1 / 1 2 / 1 2 3)

for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= i; j++) {
        System.out.print(j + " ");
    }
    System.out.println();
}

💡 Logic: What happens?

Similar to triangle, but prints the inner counter j. Since j resets to 1 every row, it always starts from 1.

❓ Requirement Change: What if...?

Change: Reverse the numbers (3 2 1 / 2 1 / 1).
Do: Change inner loop to for(int j=i; j>=1; j--).

🕒 Session 4: More Patterns + Math

11. Floyd's Triangle

int count = 1;
for (int i = 1; i <= 4; i++) {
    for (int j = 1; j <= i; j++) {
        System.out.print(count++ + " ");
    }
    System.out.println();
}

💡 Logic: What happens?

A running counter that NEVER resets. It keeps incrementing across all rows and columns.

❓ Requirement Change: What if...?

Change: Start from 0 instead of 1.
Do: Initialize count = 0.

🚀 Try Code

12. Pascal's Triangle (5 rows)

for (int i = 0; i < 5; i++) {
    int num = 1;
    for (int j = 0; j <= i; j++) {
        System.out.print(num + " ");
        num = num * (i - j) / (j + 1);
    }
    System.out.println();
}

💡 Logic: What happens?

Uses the formula for combinations nCr. Each term is derived from the previous term in the same row.

❓ Requirement Change: What if...?

Change: Print centered like a real triangle.
Do: Add a space-loop for(int s=0; s<5-i; s++) print(" "); before j loop.

13. Tricky Operators (Unary + Arithmetic)

int a = 10, b = 5;
int res = a++ + ++a * --b - b--;
System.out.println("a:"+a+" b:"+b+" res:"+res);

💡 Logic: What happens?

Follows Precedence (Postfix > Unary > Multiplicative > Additive). Post-increment changes value AFTER use, pre-increment changes BEFORE use.

❓ Result Analysis

Step: 10 (a=11) + 12 (a=12) * 4 (b=4) - 4 (b=3).
Output: a:12 b:3 res:54.

14. Armstrong Number (153, 1634, etc.)

int n = 153, temp = n, sum = 0;
int k = String.valueOf(n).length();
while(temp > 0) {
    int d = temp % 10;
    sum += Math.pow(d, k);
    temp /= 10;
}
System.out.println(n + (sum==n ? " is " : " is not ") + "Armstrong");

💡 Logic ( Narcissistic Numbers)

An Armstrong number is the sum of its digits each raised to the power of the number of digits.

Number Digits (k) Calculation Result
153 3 1³ + 5³ + 3³ 1 + 125 + 27 = 153
1634 4 1⁴ + 6⁴ + 3⁴ + 4⁴ 1 + 1296 + 81 + 256 = 1634

15. Strong Number (145 = 1! + 4! + 5!)

int n=145, temp=n, sum=0;
while(temp > 0) {
    int d = temp % 10, fact=1;
    for(int i=1; i<=d; i++) fact *= i;
    sum += fact;
    temp /= 10;
}
System.out.println(n + (sum==n ? " is " : " is not ") + "Strong");

💡 Logic

Sum of factorials of digits. If sum matches original, it's a Strong number.

16. Palindrome Number (121 → 121)

int n=121, temp=n, rev=0;
while(temp > 0) {
    rev = rev * 10 + (temp % 10);
    temp /= 10;
}
System.out.println(n + (rev==n ? " is " : " is not ") + "Palindrome");

💡 Logic

Uses digit extraction to reverse the number. If original == reversed, it's a Palindrome.

17. Perfect Number (6 = 1+2+3)

int n=6, sum=0;
for(int i=1; i < n; i++) {
    if(n % i == 0) sum += i;
}
System.out.println(n + (sum==n ? " is " : " is not ") + "Perfect");

💡 Logic

Sum of all proper divisors (excluding itself). If sum == n, it's a Perfect number.

20. Summarizing Loops: for vs while vs do-while

Loop Type Best Used For... Check Type
for Fixed range, counting, array traversal. Entry-controlled
while Unknown iterations, event-driven, condition-based. Entry-controlled
do-while User menus, running code AT LEAST once. Exit-controlled
