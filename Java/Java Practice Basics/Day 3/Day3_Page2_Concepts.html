1. Pointers — How Java References Work

Java doesn't have C-style pointers. Instead, reference variables on the Stack store the memory address of an object on the Heap.

When two keys hash to the same index, Java stores them as a linked list within the same bucket. On lookup, it scans the chain until it finds the matching key.

HashMap vs Hashtable

Feature HashMap Hashtable
Thread-safe ❌ No ✅ Yes (synchronized)
Null keys ✅ 1 null key allowed ❌ Not allowed
Performance ⭐ Faster Slower (sync overhead)
Introduced in Java 1.2 Java 1.0 (Legacy)
Use when? Single thread / modern code Legacy or multi-thread (use ConcurrentHashMap instead)
§3 · Multithreading

3. Multithreading — Parallel Execution

A Thread is a lightweight unit of execution. Java can run multiple threads concurrently — imagine multiple chefs cooking different dishes simultaneously.

🔵 Thread 1 — downloading data
🟣 Thread 2 — processing data
🟢 Thread 3 — saving to disk
Concept Description
Thread Independent path of execution inside a program
synchronized Prevents race conditions — only one thread at a time
Runnable Interface to define a task for a thread
start() Begins thread execution (not run() directly!)
join() Wait for a thread to finish before proceeding
